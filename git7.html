Contribuições e Pull Requests


Como fazer um fork de um repositório?

Para fazer um fork de um repositório, acesse a página do repositório no GitHub que deseja forkar e clique no botão "Fork" no canto superior direito. Isso criará uma cópia do repositório no seu perfil, permitindo que você faça mudanças sem afetar o repositório original.


O que é um pull request e como criar um pull request?

Um pull request é uma solicitação para que suas alterações em um repositório sejam revisadas e integradas ao código principal. Para criar um pull request, depois de fazer commit das alterações em uma branch, vá até a página do repositório no GitHub, clique em "Pull Requests", e então em "New Pull Request". Compare as branches e, se tudo estiver correto, clique em "Create Pull Request".


Como revisar e aceitar um pull request?

Para revisar um pull request, acesse a aba de pull requests no repositório, selecione o pull request que deseja revisar, e analise as mudanças propostas. Você pode deixar comentários, sugerir alterações ou aprovar o pull request. Para aceitar, clique em "Merge Pull Request" e depois em "Confirm Merge".
